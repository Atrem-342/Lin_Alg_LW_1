# Сравнение методов решения СЛАУ

Проект реализует и сравнивает методы решения систем линейных алгебраических уравнений:

- метод Гаусса без выбора главного элемента;
- метод Гаусса с частичным выбором главного элемента по столбцу;
- LU-разложение;
- прямую и обратную подстановки для решения треугольных систем.

Также в проекте есть:

- генерация случайных матриц и векторов с фиксированным `seed`;
- генерация матрицы Гильберта;
- вычисление невязки и относительной погрешности;
- замер времени работы алгоритмов;
- минимальные тесты корректности.

## Режимы работы

В `matrix_app` доступны:

- ручной ввод и решение одной системы;
- эксперимент со случайными матрицами и замером времени;
- эксперимент с несколькими правыми частями;
- эксперимент на матрице Гильберта для проверки точности.

## Структура проекта

- `main.c` - меню, запуск экспериментов и вывод результатов;
- `gauss.c/.h` - реализация методов Гаусса;
- `lu.c/.h` - LU-разложение и решение треугольных систем;
- `generate_matrix.c/.h` - генерация тестовых матриц и векторов;
- `matrix.c/.h` - простая структура квадратной матрицы `double`;
- `analysis.c/.h` - невязка и относительная погрешность;
- `clock_get_time.c/.h` - измерение времени;
- `tests.c` - проверки корректности алгоритмов.

## Формулы методоа решения СЛАУ

- Гаусс без выбора главного элемента

    A * x = b, где A — квадратная матрица размера n.

    Прямой ход (приведение к верхнетреугольному виду):

    Для k = 1, 2, ..., n-1:
        Для i = k+1, ..., n:
            множитель: m_ik = a_ik / a_kk
            Для j = k+1, ..., n:
                a_ij = a_ij - m_ik * a_kj
            b_i = b_i - m_ik * b_k

    После прямого хода получается система U * x = b', где U — верхнетреугольная матрица.

    Обратный ход (нахождение неизвестных):

    x_n = b_n / a_nn
    Для i = n-1, ..., 1:
        x_i = (b_i - сумма_{j=i+1}^{n} (a_ij * x_j)) / a_ii


- Гаусс с выбором главного элемента

    A * x = b, где A — квадратная матрица размера n.

    Прямой ход с выбором главного элемента:

    Для k = 1, 2, ..., n-1:
        // Шаг 1: Поиск главного элемента в k-м столбце
        Найти индекс p (k <= p <= n), такой что |a_pk| = максимальное среди |a_ik| для i = k..n
    
        // Шаг 2: Перестановка строк (если p != k)
        Поменять местами строки k и p в матрице A и векторе b
    
        // Шаг 3: Исключение переменных
        Для i = k+1, ..., n:
           множитель: m_ik = a_ik / a_kk
           Для j = k+1, ..., n:
               a_ij = a_ij - m_ik * a_kj
            b_i = b_i - m_ik * b_k

    После прямого хода получается система U * x = b', где U — верхнетреугольная матрица.

    Обратный ход (нахождение неизвестных):

    x_n = b_n / a_nn
    Для i = n-1, ..., 1:
        x_i = (b_i - сумма_{j=i+1}^{n} (a_ij * x_j)) / a_ii

- LU-разложение 
    Для квадратной матрицы A LU-разложение представляется в виде:

    A = L * U

    где:
    - L — нижняя треугольная матрица с единицами на главной диагонали
    - U — верхняя треугольная матрица

    Для i <= j (верхняя треугольная часть):
    u_ij = a_ij - сумма_{k=1}^{i-1} (l_ik * u_kj)

    Для i > j (нижняя треугольная часть):
    l_ij = (1 / u_jj) * (a_ij - сумма_{k=1}^{j-1} (l_ik * u_kj))

## Выводы

- Эксперимент 1. На случайных матрицах все три метода показывают близкое время работы и очень малые значения невязки и относительной ошибки. Для одной системы метод LU немного проигрывает Гауссу из-за дополнительного этапа разложения, поэтому его преимущество здесь ещё не проявляется.

- Эксперимент 2. При решении нескольких систем с одной и той же матрицей LU-разложение заметно эффективнее. Для k = 1 оно медленнее Гаусса, но уже при k = 10 и особенно при k = 100 суммарное время LU становится значительно меньше, так как разложение выполняется один раз, а затем для каждой правой части выполняются только треугольные подстановки.

- Эксперимент 3. На матрице Гильберта видно, что с ростом n относительная погрешность резко увеличивается у всех методов, хотя невязка остаётся очень малой. Это означает, что система плохо обусловлена, и даже при малой невязке решение может сильно отличаться от точного. Выбор главного элемента не устраняет эту проблему полностью, но в целом повышает численную устойчивость метода Гаусса.
